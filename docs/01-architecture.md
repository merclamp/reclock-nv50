## Где сейчас «проседает» — на основе измерений

| Подсистема | Статус на машине | Действие |
|---|---|---|
| `dispnv50` (KMS) | ✅ работает (Wayland идёт) | не трогаем |
| TTM/GEM/gpuvm | ✅ работает (direct rendering) | не трогаем |
| FIFO/gr (PGRAPH) | ✅ GL 3.3/ES 3.0 идут | не трогаем |
| **`clk` + `fb/ram` + `volt`** | ❌ сидит на AC 500/1250/499, не поднимается | **главный фронт** |
| `therm` | частично (temp читается, fan не виден) | сопутствующее |

## Reclocking: где живёт код и почему он опасен

Подсистема в kernel-дереве `[проверено]`:
`drivers/gpu/drm/nouveau/nvkm/subdev/` →
- `clk/` — расчёт и программирование PLL (ядро/шейдер/память), таблица pstate.
  Для NV50-семейства смотреть `clk/nv50.c`, `clk/base.c`, `clk/gt215.c` `[паттерн]`.
- `fb/ram*` — **переключение частоты памяти**: ввод VRAM в self-refresh, смена MPLL,
  пересчёт таймингов, retraining. Это самая хрупкая часть на NV50 `[паттерн]`.
- `volt/` — изменение напряжения перед/после смены частоты `[паттерн]`.
- `therm/` — вентилятор/защита.

Последовательность смены memory clock (укрупнённо, **сверять с исходником и rnndb**) `[паттерн]`:
```
1. поднять напряжение (volt) при росте частоты
2. остановить обращения к VRAM, перевести чипы в self-refresh
3. перепрограммировать MPLL на новую частоту
4. перенастроить тайминги/калибровку (drive strength, DLL), retrain
5. вывести из self-refresh, разморозить трафик
6. при падении частоты — снизить напряжение в конце
```
Любой неверный шаг 2–5 → чёрный экран/зависание (промпт, раздел «Безопасность»).

**Источники адресов (НЕ выдумывать):** envytools `rnndb/` —
`memory/` (контроллер памяти/MPLL), `pll.xml`, и соответствующий nvkm-исходник.
До получения исходников и rnndb конкретные регистры **не приводим**.

## Пути для игр (соответствует промпту, без изменений архитектуры)

- **Gallium Nine** (D3D9 напрямую в Gallium nouveau) — минимум накладных, лучший для эпохи NV50.
- **WineD3D** (D3D9/10/11 → OpenGL) — шире охват, медленнее.
- DXVK/Zink — неприменимы (нужен Vulkan, которого нет).
- Узкое место производительности игр здесь — **не транслятор, а низкий memory clock** ⇒
  смотри reclocking, а не замену state-tracker'а.

# 07 — Второй фронт: TTM validate -12 (ENOMEM). Предварительный разбор

Статус: предварительный. Точный корень требует рантайм-логов (⇒ присутствие пользователя).
Всё ниже — безопасный разбор исходника `src/linux/.../nouveau`.

## Где возникает (подтверждено по исходнику)

`drivers/gpu/drm/nouveau/nouveau_gem.c`:
```
validate_list()          :567
  nouveau_bo_validate(nvbo, true, false)   :587   → возвращает -12
  if (ret) "fail ttm_validate"             :589
nouveau_gem_pushbuf_validate()             :646   "validating bo list"
nouveau_gem_pushbuf_ioctl()                :829   "validate: -12"  → goto out_prevalid
```
`-12 = -ENOMEM`: TTM не смог разместить BO в запрошенном домене (VRAM) даже после
вытеснения. На ENOMEM ioctl **не ретраит** (label `revalidate:` — только для relocs/restart).

## Картина на машине (dmesg)

- 30× подряд от `Xwayland[766]`, разрешение всего **1360×768**, VRAM **512 МБ**,
  GART **1 ТБ** (т.е. fallback в системную память теоретически есть).
- Рядом 12× `gr: DATA_ERROR [INVALID_VALUE] class 8297 (G82_TESLA) mthd 0204` от того же
  Xwayland — возможно связаны (плохая 3D-команда после неудачной валидации/релока).

## Гипотезы (требуют проверки рантаймом)

1. BO, обязанный лежать в VRAM (scanout/tiled), не влезает из-за **фрагментации** VRAM,
   а placement не разрешает GART-fallback для этого буфера. `[предположение]`
2. Утечка/накопление VRAM-аллокаций под нагрузкой (KDE/Xwayland). `[предположение]`
3. Связь с reclocking маловероятна — это память-менеджмент, не частоты. `[вывод]`

## Безопасные следующие шаги (когда вернётся пользователь)

- Поднять отладку: `nouveau.debug=MMU=debug,BO=debug` или модульные параметры, повторить
  сценарий, увидеть **какой BO/размер/домен** падает.
- Снять карту VRAM: `/sys/kernel/debug/dri/0000:01:00.0/` (`gpuvas`, `gem_names`) при ошибке.
- Проверить, не специфично ли для конкретного приложения (Xwayland) или общесистемно.

## Зависимость от reclocking-фронта

Независимы. Этот фронт можно вести параллельно, он НЕ требует пропатченного модуля.
Приоритет — на усмотрение пользователя (reclocking даёт FPS, ENOMEM даёт стабильность).
